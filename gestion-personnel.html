Gestion du Personnel M2L

Application Java de gestion des employés et ligues sportives

Janvier 2024 Java • MySQL • JDBC BTS SIO SLAM

Contexte du projet

Ce projet a été réalisé dans le cadre de ma formation BTS SIO option SLAM. L'objectif était de développer une application de gestion du personnel pour la Maison des Ligues de Lorraine (M2L), permettant de gérer les ligues sportives et leurs employés avec une base de données MySQL.

Modèle Conceptuel de Données (MCD)

Relations entre entités

APPARTENIR
LIGUE (1,n) ←→ EMPLOYE (0,1)
ADMINISTRER
LIGUE (0,1) ←→ EMPLOYE (0,1)

APPARTENIR : Un employé appartient à 0 ou 1 ligue. Une ligue peut avoir plusieurs employés.

ADMINISTRER : Une ligue est administrée par 0 ou 1 employé. Un employé peut administrer 0 ou 1 ligue.

Structure des tables SQL

CREATE TABLE LIGUE ( num_ligue INT PRIMARY KEY AUTO_INCREMENT, nom VARCHAR(100) NOT NULL, num_employe INT, FOREIGN KEY (num_employe) REFERENCES EMPLOYE(num_employe) ); CREATE TABLE EMPLOYE ( num_employe INT PRIMARY KEY AUTO_INCREMENT, nom VARCHAR(100) NOT NULL, prenom VARCHAR(100) NOT NULL, mail VARCHAR(255) NOT NULL UNIQUE, password VARCHAR(255) NOT NULL, date_arrivee DATE NOT NULL, date_depart DATE, num_ligue INT, FOREIGN KEY (num_ligue) REFERENCES LIGUE(num_ligue) );
